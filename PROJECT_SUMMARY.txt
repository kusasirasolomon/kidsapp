   npm run dev
   # Open http://localhost:5173

3. BUILD (when ready):
   npm run build

4. DEPLOY (pick one):
   - Vercel: vercel
   - GitHub Pages: gh-pages -d dist
   - Traditional: Upload dist/ to web server

================================================================================
                          DATABASE SCHEMA
================================================================================

LESSONS TABLE:
  - id (uuid, primary key)
  - title (text) - Lesson title
  - video_url (text) - YouTube embed URL
  - story_text (text) - Reading story
  - vocabulary (jsonb) - Array of {word, definition}
  - order_number (integer) - Lesson order

QUIZ_RESULTS TABLE:
  - id (uuid, primary key)
  - user_id (uuid) - References auth.users
  - lesson_id (uuid) - References lessons
  - score (integer) - Number correct
  - total_questions (integer) - Total questions
  - answers (jsonb) - Array of {question, userAnswer, correctAnswer, correct}
  - created_at (timestamp)

USER_PROGRESS TABLE:
  - id (uuid, primary key)
  - user_id (uuid) - References auth.users
  - lesson_id (uuid) - References lessons
  - completed (boolean) - Whether completed
  - completed_at (timestamp) - When completed

================================================================================
                      WHAT'S DIFFERENT FROM REACT
================================================================================

ADVANTAGES OF VANILLA JS:
  ✓ No React learning curve
  ✓ Smaller bundle (9KB vs 150KB+)
  ✓ No build step needed
  ✓ Pure JavaScript anyone can understand
  ✓ Direct DOM control
  ✓ Faster development
  ✓ Easy to debug
  ✓ No framework constraints

TRADE-OFFS:
  ✗ No TypeScript type safety
  ✗ Manual state management
  ✗ More DOM manipulation code
  ✗ No component reuse patterns
  ✗ Less tooling support

BUT IT WORKS PERFECTLY FOR:
  ✓ Educational apps
  ✓ Simple applications
  ✓ Learning projects
  ✓ Rapid prototyping
  ✓ High performance needed
  ✓ Small teams
  ✓ Easy customization
